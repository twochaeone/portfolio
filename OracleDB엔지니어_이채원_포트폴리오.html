수천 개 테이블 사전 검증을 위한
Bad Data 추출 쿼리 자동 생성 시스템

문제: 본이행 전, AS-IS 스테이징 DB의 수천 개 테이블에 대해 매핑 정의서 기준 검증 규격에 어긋나는 Bad Data를 사람이 일일이 쿼리로 찾아낼 수 없는 규모.

해결 - SQL을 만드는 SQL: 매핑 정의서(자료형, 자릿수, Not Null, 코드값 등 검증 규격)를 기준으로 **각 테이블/컬럼별 Bad Data 추출 쿼리 자체를 생성하는 쿼리**를 설계, 수천 개 검증 쿼리를 자동으로 산출.

실행: 생성된 쿼리를 Shell Script로 일괄 구동, 배치 프로그램화하여 수천 개 테이블을 정기적으로 검증.

함수화: 날짜 형식 검증처럼 반복되는 로직은 `IS_DATE()` 같은 PL/SQL 사용자 정의 함수로 만들어 검증 쿼리 전반에서 재사용.

02번 검증 배치의 속도 이슈를
Unpivot 압축과 ROWID 정렬 분할로 해결

문제 1 - 속도 저하: PROJECT 02의 검증 쿼리를 계속 운영하다 보니 컬럼 단위로 흩어진 검증 결과를 모으는 과정에서 속도 이슈 발생.

해결 1 - Unpivot 압축 통합: 컬럼 단위로 흩어져 있던 검증 로직을 **Unpivot 기법으로 압축·통합하는 프로그램을 Python으로 직접 개발**, 처리 단위를 대폭 줄임.

문제 2 - ORA-01555: 대용량 테이블 검증 시 ORA-01555 (snapshot too old)가 반복 발생.

해결 2 - ROWID 분할 + 물리 정렬: 작업을 **ROWID 범위로 나눠 분할 처리**하고, 이때 **물리적 블록 주소가 가까운 로우끼리 ORDER BY** 하여 랜덤 I/O를 줄이고 메모리 사용을 최소화.

OS-DB 자원 관제

CPU 터지면 범인 PID 역추적

`top`, `vmstat`을 통해 CPU 과부하 탐지 즉시, 특정 PID를 인프라 백엔드의 오라클 내부 세션 세부 정보(`V$SESSION`, `V$PROCESS`)와 수동 매핑(SPID). 부서별 유발 프로세스를 역추적해 악성 쿼리를 통제 유도.

옵티마이저 왜곡 해결

E-Rows vs A-Rows 편차 교정

실행계획 상 비용 예측치($E-Rows$)와 실제 로우 건수($A-Rows$)의 불일치를 탐지하고, `DBMS_STATS` 패키지를 통해 낡은 통계 정보 갱신 및 특수 조인 힌트를 적용하여 옵티마이저 오류를 즉각 복원 및 튜닝함.

스토리지 선제 조치

Tablespace 데이터파일 할당

초대용량 적재 중 파일 용량 임계치 오버플로우가 나기 직전, `ALTER TABLESPACE ... ADD DATAFILE` 혹은 `AUTOEXTEND ON`을 신속하게 조작하여 무정지 이행 스토리지 확장 안정성을 직접 확보해 봄.

Java 백엔드 커넥션

JDBC 접속 끊김 소스 디버깅

Java 기반의 적재용 툴의 DB 링크 연동 및 세션 타임아웃에 의한 접속 단절 현상에 대처하기 위해, 백엔드 설정 영역을 열어 JDBC 커넥션 풀 정보 및 연동 타임아웃 파라미터를 역추적하고 직접 패치함.

분석 프로세스: 전력 사용량 시계열 데이터를 뜯어, 정확도($R^2$)를 최적화하기 위해 데이터 전처리 및 상관관계 기반의 차원 축소를 적용한 머신러닝 분석.

다중공선성(Multicollinearity) 해결: 타겟 변수($y$)와의 연계성은 높이고, 피쳐 간에 연관도가 너무 높은 변수(다중공선성 리스크)는 히트맵 시각화를 통해 발라내고 제거하여 학습 코스트 최소화.

주간 주기성(Lag Feature) 발굴: MATLAB 시각화 중 전력 데이터가 '일주일 주기(Seasonality)'로 완벽한 규칙성을 띠는 패턴 발견. **'일주일 전 동 시간대 사용량'을 Lag Feature로 추가**하여 예측도 상승.

SQL Generates SQL

수천 개 테이블 사전 검증을 위한
Bad Data 추출 쿼리 자동 생성 시스템

문제: 본이행 전, AS-IS 스테이징 DB의 수천 개 테이블에 대해 매핑 정의서 기준 검증 규격에 어긋나는 Bad Data를 사람이 일일이 쿼리로 찾아낼 수 없는 규모.

해결 - SQL을 만드는 SQL: 매핑 정의서(자료형, 자릿수, Not Null, 코드값 등 검증 규격)를 기준으로 **각 테이블/컬럼별 Bad Data 추출 쿼리 자체를 생성하는 쿼리**를 설계, 수천 개 검증 쿼리를 자동으로 산출.

실행: 생성된 쿼리를 Shell Script로 일괄 구동, 배치 프로그램화하여 수천 개 테이블을 정기적으로 검증.

Python으로 전환: 이후 검증 대상이 SQL*Plus의 LINESIZE 한계(32,767자)를 넘는 대용량 CLOB SQL로 확장되면서, 쿼리 생성·파일 출력 로직을 Shell에서 Python 스크립트로 직접 재작성해 대응. 이 과정에서 인코딩(UTF-8), 개행 문자, 경로 처리 등 실무 파일 자동화 노하우를 축적.

함수화: 날짜 형식 검증처럼 반복되는 로직은 `IS_DATE()` 같은 PL/SQL 사용자 정의 함수로 만들어 검증 쿼리 전반에서 재사용.

확장 가능성: 검증 규격을 기준으로 데이터 오류를 정기적으로 자동 탐지·기록하는 구조라, 데이터 정합성에 대한 내부통제(ITGC) 관점의 상시 모니터링 체계로도 적용할 수 있습니다.

PROJECT 02. ETL 메타 DB 역공학 자동화 (MySQL)

ETL 툴 메타 DB 역공학 분석을 통한
수천 개 이행 작업 자동화 생성

상황: 신규 매핑 정의서에 맞춰 수천 개의 데이터 전환 작업 생성 자동화 프로그램 필요

역공학(Reverse Engineering): DB 스키마 구조 문서가 전혀 없는 폐쇄적 환경. UI로샘플 작업을 하나 등록한 직후, 시스템 테이블에서 변경 시각이 탐지된 물리 테이블들의 관계를 역추적.

해결: Python 스크립트로 MySQL 메타 저장소에 직접 작업을 주입하는 파이프라인 완성. **수주일 짜리 반복 작업을 단 몇 분 만에 종료.**

방법론: 문서 없는 시스템의 구조를 UI 조작 → 내부 테이블 변화 관찰 → 관계 역추적 방식으로 밝혀낸 경험은, 명세 없는 웹사이트의 구조를 분석하고 자동화 대응 로직을 설계하는 작업과 접근 방식이 동일하다고 생각합니다.
